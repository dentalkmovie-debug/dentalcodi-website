## 1. 개요

영상 배포 시스템은 **최고관리자(마스터)**가 광고/홍보 영상을 각 치과의 대기실/체어 TV 재생목록에 **자동으로 삽입**하는 기능입니다.

### 핵심 기능
- YouTube/Vimeo 영상을 치과 TV에 원격 배포
- 전체 치과 또는 선택 치과에 배포
- 대기실/체어/전체 TV 타입 지정
- **3가지 삽입 위치**: 맨 뒤, 맨 앞, 균등 배치
- **반복 재생 시 every:N** 삽입 지원
- 드래그 앤 드롭으로 재생 순서 변경
- **치과별 개별 관리**: 광고 제거, 순서 변경
- 자동 만료 설정

## 2. 배포 화면 접근

### 마스터 관리자 페이지
1. **관리자** 페이지 접속
2. 상단 탭에서 **관리** 클릭
3. 서브 탭에서 **영상 배포** 클릭

### 아임웹 관리자 페이지
1. 아임웹 관리 페이지 접속
2. 서브 탭에서 **영상 배포** 클릭

## 3. 새 영상 배포하기

### 입력 항목

| 항목 | 설명 | 예시 |
|------|------|------|
| **URL** | YouTube 또는 Vimeo 영상 주소 | `https://vimeo.com/123456789` |
| **배포 제목** | 광고 이름 (관리용) | `3월 신제품 광고` |
| **대상** | 전체 치과 / 선택 치과 | 전체 치과 |
| **삽입 위치** | 맨 뒤 / 맨 앞 / 균등 배치 | 균등 배치 |
| **대상 TV** | 전체 / 대기실만 / 체어만 | 전체 |
| **자동 만료 (일)** | 0 = 무기한, 숫자 입력 시 해당 일 후 자동 종료 | `30` |

### 배포 절차
1. URL 입력
2. 제목 입력
3. 대상, 삽입 위치, TV 타입 선택
4. (선택) 자동 만료일 설정
5. **배포하기** 버튼 클릭
6. "N개 치과에 배포 완료!" 메시지 확인

## 4. 삽입 위치 3가지 모드

### 4-1. 맨 뒤 (end)
```
치과 영상: [V1] [V2] [V3] [V4] [V5]
광고 삽입: [V1] [V2] [V3] [V4] [V5] [광고A]
```
- 가장 기본적인 모드
- 치과 영상이 모두 재생된 후 광고가 나옴
- 반복 재생에서도 동일한 위치

### 4-2. 맨 앞 (start)
```
치과 영상: [V1] [V2] [V3] [V4] [V5]
광고 삽입: [광고A] [V1] [V2] [V3] [V4] [V5]
```
- TV가 켜지면 가장 먼저 광고가 재생
- 시선 집중 효과가 높음
- 반복 재생에서도 동일한 위치

### 4-3. 균등 배치 (ratio:auto)
```
치과 영상: [V1] [V2] [V3] [V4] [V5] [V6]
광고 1개:  [V1] [V2] [V3] [광고A] [V4] [V5] [V6]
광고 2개:  [V1] [V2] [광고A] [V3] [V4] [광고B] [V5] [V6]
광고 3개:  [V1] [광고A] [V2] [V3] [광고B] [V4] [V5] [광고C] [V6]
```
- 치과 영상 사이에 광고를 균등하게 분배
- 시청자가 자연스럽게 광고를 접함
- **"반복 재생 시 N개 영상마다 삽입" 옵션 활성화 가능**

## 5. 균등 배치 상세 설명 (ratio:auto)

### 균등 배치 알고리즘

1. 전체 치과 영상 수를 센다 → `regularCount`
2. 균등 배치 광고 수를 센다 → `adCount`
3. 간격 계산: `interval = Math.floor(regularCount / (adCount + 1))`
4. 최소 간격: 2 이상 보장 (광고가 연속되지 않도록)

### 계산 예시

#### 예시 1: 영상 9개 + 광고 2개
```
interval = Math.floor(9 / (2 + 1)) = Math.floor(3) = 3

위치: 3번째 뒤, 6번째 뒤

결과: [V1] [V2] [V3] [광고A] [V4] [V5] [V6] [광고B] [V7] [V8] [V9]
```

#### 예시 2: 영상 12개 + 광고 3개
```
interval = Math.floor(12 / (3 + 1)) = Math.floor(3) = 3

위치: 3번째 뒤, 6번째 뒤, 9번째 뒤

결과: [V1] [V2] [V3] [광고A] [V4] [V5] [V6] [광고B] [V7] [V8] [V9] [광고C] [V10] [V11] [V12]
```

#### 예시 3: 영상 5개 + 광고 1개
```
interval = Math.floor(5 / (1 + 1)) = Math.floor(2.5) = 2

위치: 2번째 뒤

결과: [V1] [V2] [광고A] [V3] [V4] [V5]
```

#### 예시 4: 영상 4개 + 광고 4개
```
interval = Math.floor(4 / (4 + 1)) = 0 → 최소값 2 적용

결과: [V1] [V2] [광고A] [V3] [광고B] [V4] [광고C] [광고D]
(간격이 부족하면 남은 광고는 뒤에 배치)
```

### 핵심 원리
- 광고끼리 연속되지 않도록 최소 간격 2 보장
- 치과 영상 수가 적으면 일부 광고가 뒤에 몰릴 수 있음
- **우선순위(재생 순서)가 높은 광고가 먼저 삽입됨**

## 6. 반복 재생 시 every:N 전략

### every:N이란?

TV는 재생목록을 끝까지 재생하면 **처음부터 다시 반복**합니다.
`every:N`은 **2번째 반복부터** N개의 일반 영상이 재생될 때마다 광고를 한 번 삽입하는 전략입니다.

### 설정 방법
1. 삽입 위치를 **"균등 배치"** 선택
2. **"반복 재생 시 N개 영상마다 삽입"** 체크박스 활성화
3. N 값 입력 (기본값: 5, 최소: 2)

### 동작 원리

```
[1차 재생] → ratio:auto 그대로 (균등 배치)
[V1] [V2] [V3] [광고A] [V4] [V5] [V6] [광고B] [V7] [V8] [V9]

[2차 재생부터] → every:N 모드 (N=3일 때)
[V1] [V2] [V3] [광고A] [V4] [V5] [V6] [광고B] [V7] [V8] [V9] [광고A] ...
  1     2     3   → 삽입!   4     5     6   → 삽입!   7     8     9   → 삽입!
```

### every:N 카운터 규칙
- 일반 영상(치과 영상)이 재생될 때만 카운터 증가
- 광고가 재생될 때는 카운터가 증가하지 않음
- 카운터가 N에 도달하면 → 광고 삽입 → 카운터 리셋

## 7. ratio:auto + every:N 조합 동작

이것이 이 시스템의 핵심입니다. 두 전략이 단계적으로 적용됩니다.

### 단계별 동작

```
┌─────────────────────────────────────────────────────────┐
│                    1차 재생 (첫 번째 루프)                  │
│                                                          │
│  서버가 ratio:auto로 계산한 위치에 광고가 배치됨             │
│  [V1][V2][V3][광고A][V4][V5][V6][광고B][V7][V8][V9]       │
│                                                          │
│  → 재생목록 그대로 순서대로 재생                            │
├─────────────────────────────────────────────────────────┤
│                    2차 재생 이후 (반복 루프)                 │
│                                                          │
│  every:N 모드 활성화 (N=3인 경우)                          │
│  일반 영상 3개마다 광고 1개 자동 삽입                        │
│                                                          │
│  [V1][V2][V3] → [광고A] → [V4][V5][V6] → [광고B] → ...   │
│     1    2   3   삽입!     4    5    6    삽입!            │
└─────────────────────────────────────────────────────────┘
```

### 왜 2단계 구조인가?
- **1차 재생**: 방문 환자가 처음 보는 화면이므로, 균등 배치로 자연스러운 경험
- **2차+ 재생**: 장시간 대기 환자를 위해, 규칙적인 간격으로 반복 노출

### 다중 광고 라운드 로빈
every:N에서 광고가 여러 개면 **순서대로 돌아가며** 삽입됩니다:

```
광고 목록: [광고A, 광고B, 광고C]  N=4

[V1][V2][V3][V4] → 광고A → [V5][V6][V7][V8] → 광고B → [V9][V1][V2][V3] → 광고C → [V4][V5][V6][V7] → 광고A → ...
```

## 8. 재생 순서 드래그 변경 (전체)

### 기능 설명
배포된 광고가 여러 개일 때, **드래그 앤 드롭**으로 재생 순서를 변경할 수 있습니다.
이 순서는 **전체 치과에 공통으로** 적용됩니다.

### 사용 방법
1. 배포 현황 목록에서 활성 광고 확인
2. 각 광고 왼쪽의 **⠿ (드래그 핸들)** 아이콘을 잡고 끌기
3. 원하는 위치에 놓기
4. 순서 번호가 자동 갱신됨
5. 서버에 자동 저장됨

### 순서가 적용되는 곳
| 적용 영역 | 설명 |
|-----------|------|
| **균등 배치 순서** | 1번 광고가 먼저 삽입됨 |
| **every:N 라운드 로빈** | 1번 → 2번 → 3번 순서로 재생 |
| **맨 앞/맨 뒤 정렬** | 1번이 가장 앞/뒤에 위치 |

### 예시
```
순서 변경 전: [광고A(1번)] [광고B(2번)] [광고C(3번)]
드래그로 변경: [광고C(1번)] [광고A(2번)] [광고B(3번)]

→ 균등 배치 시: 광고C가 첫 번째 삽입 지점에 배치
→ every:N 시: 광고C → 광고A → 광고B 순서로 재생
```

## 9. 치과별 개별 관리

### 개요
A치과에 2개 영상, B치과에 10개 영상을 보낸 경우, **각 치과별로 독립적으로** 광고를 관리할 수 있습니다.

### 치과별 배포 현황 확인
배포 현황 아래에 **"치과별 배포 현황"** 섹션이 있습니다.

```
┌────────────────────────────────────────┐
│  치과별 배포 현황          [새로고침]    │
├────────────────────────────────────────┤
│  > 로이스치과    3개 광고   대기실(12)   │
│  > 강남미소치과  2개 광고   체어(8)      │
│  > 교동치과      5개 광고   대기실(15)   │
└────────────────────────────────────────┘
```

### 치과 패널 펼치기
치과명을 **클릭**하면 해당 치과에 배포된 광고 목록이 펼쳐집니다.

```
┌────────────────────────────────────────┐
│  v 로이스치과    3개 광고               │
│  ┌──────────────────────────────────┐  │
│  │ ⠿ 1  [썸네일] 덴티스 광고  균등  │  │
│  │ ⠿ 2  [썸네일] 가이스트리히  맨뒤  │  │
│  │ ⠿ 3  [썸네일] 3월 프로모션  맨앞  x │ │
│  └──────────────────────────────────┘  │
│  > 강남미소치과  2개 광고               │
└────────────────────────────────────────┘
```

### 치과별 드래그 순서 변경
1. 치과 패널을 **펼치기** (치과명 클릭)
2. 각 광고의 **⠿ 드래그 핸들**을 잡고 순서 변경
3. 순서가 **해당 치과에서만** 독립적으로 변경됨
4. 다른 치과의 순서에는 영향 없음

```
[예시: 로이스치과에서만 순서 변경]

변경 전: 1. 덴티스  2. 가이스트리히  3. 프로모션
드래그:   1. 프로모션  2. 덴티스  3. 가이스트리히

→ 로이스치과 TV에서만 프로모션이 먼저 재생
→ 다른 치과는 원래 순서 유지
```

### 치과별 광고 제거
1. 치과 패널을 펼침
2. 제거할 광고의 **x (제거)** 버튼 클릭
3. 확인 팝업에서 "확인" 클릭
4. **해당 치과에서만** 광고가 제거됨 (다른 치과는 유지)

```
[예시: A치과에 10개 광고 배포 후 8개 제거]

배포 직후: A치과 10개, B치과 10개
제거 후:   A치과 2개, B치과 10개 (그대로)
```

### 전체 순서 vs 치과별 순서
| 구분 | 전체 순서 (배포 현황) | 치과별 순서 (치과별 현황) |
|------|---------------------|------------------------|
| **범위** | 모든 치과에 공통 적용 | 해당 치과에만 적용 |
| **우선순위** | 기본값으로 사용 | 전체 순서보다 우선 |
| **변경 위치** | 배포 현황 카드 | 치과별 현황 → 치과 펼치기 |
| **용도** | 기본 재생 순서 설정 | 특정 치과 맞춤 순서 |

## 10. 치과별 관리 시나리오

### 시나리오 1: A치과 2개, B치과 10개
```
[1단계] 10개 광고를 "전체 치과"에 배포
→ A치과: 10개, B치과: 10개

[2단계] 치과별 현황에서 A치과 펼치기
→ 불필요한 8개 광고를 x 버튼으로 제거

[결과] A치과: 2개, B치과: 10개
→ 각각 독립적으로 드래그 순서 변경 가능
```

### 시나리오 2: 치과마다 다른 순서
```
A치과 재생 순서: 프로모션 → 덴티스 → 가이스트리히
B치과 재생 순서: 가이스트리히 → 프로모션 → 덴티스

→ 각 치과 패널에서 개별 드래그로 설정
```

### 시나리오 3: 특정 치과에만 추가 광고
```
[1단계] 기본 3개 광고를 "전체 치과"에 배포
[2단계] VIP치과 전용 광고 2개를 "선택 치과" → VIP치과만 선택하여 배포

[결과] 
일반 치과: 3개 광고
VIP치과: 5개 광고 (기본 3 + 전용 2)
→ VIP치과에서 5개의 순서를 드래그로 조정
```

### 시나리오 4: 점진적 배포
```
[1주차] 3개 치과에만 테스트 배포 → "선택 치과" 모드
[2주차] 반응 좋으면 "전체 치과"로 추가 배포
[3주차] 반응 나쁜 치과는 치과별 현황에서 제거
```

## 11. 10개 광고 배포 시나리오

### 시나리오: 치과 영상 20개, 광고 10개

#### 전략 A: 전부 균등 배치
```
설정: 10개 모두 insert_position = "균등 배치"

결과:
interval = Math.floor(20 / 11) = 1

→ 영상 2개마다 광고 1개씩 삽입
[V1][V2][광고1][V3][V4][광고2][V5][V6][광고3]...[V19][V20][광고10]
```

#### 전략 B: 중요도별 혼합
```
광고 1~2 (프리미엄): 맨 앞 → TV 시작 시 바로 노출
광고 3~6 (일반):     균등 배치 + every:3 → 영상 사이에 배치, 반복 시 3개마다
광고 7~10 (보조):    맨 뒤 → 치과 영상 후 재생

결과 (1차 재생):
[광고1][광고2] [V1][V2][V3][광고3][V4][V5][V6][광고4][V7][V8][V9][광고5]
[V10][V11][V12][광고6] [V13]...[V20] [광고7][광고8][광고9][광고10]
```

#### 전략 C: 전부 균등 배치 + every:5
```
설정: 10개 모두 균등 배치 + every:5

1차 재생: 균등 분배
2차 재생: 일반 영상 5개마다 광고 (10개 라운드 로빈)

[V1][V2][V3][V4][V5] → 광고1 → [V6][V7][V8][V9][V10] → 광고2 → ...
```

### 권장 전략
| 상황 | 권장 설정 |
|------|----------|
| 광고 1~3개 | 균등 배치 |
| 광고 4~6개 | 균등 배치 + every:4~5 |
| 광고 7~10개 | 프리미엄은 맨 앞, 나머지는 균등 배치 + every:5 |
| 광고 > 10개 | 분류하여 혼합 전략 사용 |

## 12. 배포 관리 (취소/만료)

### 배포 취소
1. 배포 현황에서 취소할 광고의 **X** 버튼 클릭
2. 확인 팝업에서 "확인" 클릭
3. 상태가 "취소됨"으로 변경
4. 다음 TV 데이터 갱신 시 광고가 제거됨

### 자동 만료
- 배포 시 **자동 만료 (일)** 설정 가능
- 예: 30 입력 → 30일 후 자동으로 비활성화
- 0 = 무기한 (수동 취소 필요)

### 상태 배지 의미
| 배지 | 의미 |
|------|------|
| 활성 (녹색) | 현재 TV에서 재생 중 |
| 취소됨 (회색) | 관리자가 취소함 |
| 균등 (주황) | 균등 배치 모드 |
| 맨앞 (회색) | 맨 앞 삽입 |
| 맨뒤 (회색) | 맨 뒤 삽입 |
| 매N개 (주황 텍스트) | every:N 설정됨 |
| 수신: X/Y | Y명 중 X명이 수신 완료 |

## 13. 요약 정리표

| 기능 | 설명 |
|------|------|
| **맨 뒤** | 치과 영상 끝에 고정 삽입 |
| **맨 앞** | 치과 영상 앞에 고정 삽입 |
| **균등 배치** | 치과 영상 사이에 균등 분배 |
| **every:N** | 2차 반복부터 N개 영상마다 광고 삽입 |
| **조합** | 1차=균등 배치, 2차+=every:N |
| **전체 드래그 순서** | 모든 치과 공통 재생 순서 변경 |
| **치과별 드래그 순서** | 특정 치과만의 독립 재생 순서 변경 |
| **치과별 광고 제거** | 특정 치과에서만 광고 제거 |
| **라운드 로빈** | 여러 광고를 순서대로 돌아가며 삽입 |
| **자동 만료** | 설정 일수 후 자동 비활성 |
| **대상 지정** | 전체/선택 치과, 대기실/체어/전체 |
